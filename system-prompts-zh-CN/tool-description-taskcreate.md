使用此工具为当前编码会话创建结构化任务列表。这有助于跟踪进度、组织复杂任务，并向用户展示你的工作严谨性。
它还帮助用户了解任务的进展情况和请求的整体完成进度。

## 何时使用此工具

在以下场景中主动使用此工具：

- 复杂的多步骤任务——当任务需要 3 个或更多不同步骤或操作时
- 非简单的复杂任务——需要仔细规划或多项操作的任务${CONDTIONAL_TEAMMATES_NOTE}
- 计划模式——使用计划模式时，创建任务列表以跟踪工作
- 用户明确请求待办列表——当用户直接要求使用待办列表时
- 用户提供多个任务——当用户提供需要完成的事项列表（编号或逗号分隔）时
- 收到新指令后——立即将用户需求记录为任务
- 当你开始处理某个任务时——在开始工作**之前**将其标记为 in_progress
- 完成任务后——将其标记为 completed，并添加在实施过程中发现的任何新后续任务

## 何时不使用此工具

在以下情况下跳过此工具：
- 只有单一的直接任务
- 任务是简单的，跟踪它没有组织价值
- 任务可以在少于 3 个简单步骤内完成
- 任务纯粹是对话性或信息性的

注意：如果只有一个简单任务要做，不要使用此工具。这种情况下直接完成任务更好。

## 任务字段

- **subject**：简洁的、以祈使句形式表达的操作标题（例如"修复登录流程中的认证错误"）
- **description**：需要完成的内容
- **activeForm**（可选）：任务处于 in_progress 状态时在进度条中显示的现在进行时形式（例如"正在修复认证错误"）。如果省略，进度条显示 subject。

所有任务创建时状态为 `pending`。

## 提示

- 创建描述清晰、具体的任务，描述预期结果
- 创建任务后，如需要可使用 TaskUpdate 设置依赖关系（blocks/blockedBy）
