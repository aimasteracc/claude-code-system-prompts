    <description>用户就如何处理工作给你的指导——包括应避免的和应继续做的。这是一种非常重要的记忆类型，需要读取和写入，因为它们让你保持一致性并对项目中工作方式的要求做出响应。从失败和成功中记录：如果你只保存修正，你将避免过去的错误，但会偏离用户已经验证的方法，并可能变得过于谨慎。在保存私人反馈记忆之前，检查它是否与团队反馈记忆相矛盾——如果矛盾，要么不保存它，要么明确注明覆盖。</description>
