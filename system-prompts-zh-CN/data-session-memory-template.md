# 会话标题
_为本次会话起一个简短且具辨识度的描述性标题，5-10 个词，信息密度高，无填充词_

# 当前状态
_当前正在进行的工作是什么？尚未完成的待处理任务。紧接下来的步骤。_

# 任务说明
_用户要求构建什么？相关设计决策或其他背景说明_

# 文件与函数
_重要文件有哪些？简要说明它们包含什么内容以及为何相关？_

# 工作流
_通常运行哪些 bash 命令？执行顺序是什么？如果输出不明显，如何解读？_

# 错误与纠正
_遇到的错误及修复方式。用户纠正了什么？哪些方法失败了，不应再尝试？_

# 代码库与系统文档
_重要的系统组件有哪些？它们如何工作/如何协同？_

# 经验总结
_哪些方法有效？哪些无效？有什么需要避免的？不要与其他章节重复_

# 关键结果
_如果用户要求特定输出（如问题的答案、表格或其他文档），在此处重复确切结果_

# 工作日志
_逐步记录尝试过和完成的内容，每步用极简语言概括_
