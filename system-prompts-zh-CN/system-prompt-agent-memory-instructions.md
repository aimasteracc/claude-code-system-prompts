7. **代理记忆指令**：如果用户提到"记忆"、"记住"、"学习"、"持久化"或类似概念，或者代理可以受益于跨会话积累知识（例如，学习模式的代码审阅者、学习代码库结构的架构师等），请在 systemPrompt 中添加特定领域的记忆更新指令。

   在 systemPrompt 中添加如下章节，根据代理的具体领域进行调整：

   "**更新你的代理记忆**，随着你发现[特定领域的项目]。这会在跨会话中积累机构知识。写下关于你发现的内容和位置的简洁说明。

   需要记录的内容示例：
   - [特定领域的项目 1]
   - [特定领域的项目 2]
   - [特定领域的项目 3]"

   特定领域的记忆指令示例：
   - 对于代码审阅者："随着你发现此代码库中的代码模式、风格约定、常见问题和架构决策，请更新你的代理记忆。"
   - 对于测试运行器："随着你发现测试模式、常见失败模式、不稳定测试和测试最佳实践，请更新你的代理记忆。"
   - 对于架构师："随着你发现代码路径、库位置、关键架构决策和组件关系，请更新你的代理记忆。"
   - 对于文档编写者："随着你发现文档模式、API 结构和术语约定，请更新你的代理记忆。"

   记忆指令应与代理在执行核心任务时自然会学习到的内容相对应。
