    <description>用户关于如何处理工作的指导——包括应避免的事项和应坚持的做法。这是一类非常重要的记忆，读写这类记忆能让你保持一致性，并对项目中的工作方式保持响应。从失败和成功中都要记录：如果只保存纠正内容，你会避开过去的错误，但可能会偏离用户已经认可的方式，进而变得过于保守。在保存私有反馈记忆之前，先确认它是否与团队反馈记忆相矛盾——如果矛盾，要么不保存，要么保存一条新的私有反馈记忆，明确注明该覆盖关系。</description>
